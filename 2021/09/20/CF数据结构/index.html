CF19D Points

其实是一道 SB 题，但是看到是三维问题一个 $\log$ 就没敢想。

直接离散化，接着横坐标记为序列维，纵坐标记为值域维，线段树二分可以确定横坐标，然后对于每个横坐标开个 set 就可以确定纵坐标。

其实这个东西也不能说是三维，因为它不是让你数点而是求最值。这样的问题似乎就可以很方便地做了。

CF145D Lucky Pair

十分不会做这种题，只好翻译官方题解了。

首先发现幸运数字个数不超过 $1000$，这个性质肯定要利用。

然后考虑对于固定的右线段的左端点 $i$ ，枚举这个右线段的右端点 $j$，每次加入 $a[j]$，如果它在右端点里还没出现过并且它是幸运数字，那么 $a[j]$ 就会把 $[1:i-1]$ 这段划分成若干小段，左线段只能是这些小段的子串。所以对于 $[1:i-1]$ 这段开一个 $set$ 维护连续段就可以做到 $O(n^2\log n)$。

发现 $i$​ 和 $j$​ 可以只与幸运数字有关，所以只用 $O(1000^2\log n+)$​ 的复杂度了。

但是这个东西只能处理右线段中有幸运数字的，考虑没有的怎么做。设 $F(i)$​ 表示 $i$​ 右边第一个幸运数字的位置，枚举每个非幸运数字为右线段的左端点，就可以算出不包含幸运数字的右线段的个数，此时左线段就可以任取了。

CF150E Freezing with Style

没写

看到中位数，想到 $+1-1$。

这里我们可以二分一下，然后如果存在一条合法且边权 $\ge 0$ 的路径就说明可以继续变大。

然后考虑点分治，每次扫完一棵子树就把所有信息更新到一棵线段树上，查询合法路径就是一段区间询问了。这是 $O(n\log^3n)$ 的。

然后我们又发现路径长度的限制就像一个滑动窗口，考虑单调队列。

点分治常见技巧单调队列按秩合并：把子树大小从小到大排序，依次扫描构建单调队列。

这样复杂度就是 $\sum siz$ 了，可以少一个 $\log$。

CF173E Camping Groups

首先可以很方便地用树状数组把每个人当队长时团队人数的最大值求出来。

然后把询问从大到小排序，把 $rk$ 大于两人 $rk$ 的人都插入候选集合。

接着把 $\pm k$ 的操作贡献到那两个人的上面，这样那两个人的信息还是一个区间，而候选人的信息只是单点。

这样插在线段树上维护区间最大值就好了。

一个性质——关键点的导出子树边权和。设所有关键点按 $dfn$​ 排序后的序列为 $u_1,u_2,\ldots,u_m$​，则边权和为：

这个式子的理解方式是，考虑 $dfs$ 这棵导出子树，每条边都会被恰好算到 $2$ 次。

然后直接 set 维护即可。

CF526F Pudding Monsters

法一

首先进行题意转化。考虑把矩阵拍扁，那么问题就转化为：给你一个排列，询问有多少区间满足 $\max-\min=\mathrm{len}-1$。

可以发现用单调栈单调队列维护 $\min/\max$。

然后考虑一个 $\log$ 的数据结构。这里使用线段树。

用单调栈维护一下，每次就是更新时先撤回之前的操作，然后区间根据 $\max/\min$ 变化量加减。统计答案时考虑左儿子的 $\max-\min-\mathrm{len}$ 是不是和当前节点的相同，如果相同就累加进贡献里。显然 $1$ 号（根节点）的 $\max-\min-\mathrm{len}=-1$，因为有单点存在。然后每次枚举都加入贡献就行了。

法二

分治。

首先 $\min,\max$ 都在一边的情况可以用前后缀 $\max/\min$ 求出。

然后考虑 $\min/\max$ 分别在两边的情况。以最小值在左边为例。

那么随着我左端点向左，假如扫到的这个值是新的最小值，那么显然满足当前的右边的 $\max$ 大于左边的 $\min$，如果这个值比右边的 $\max$ 还大，那就直接让右端点向右扫。

因为满足条件的区间符合 $\min(l)-l=\max(r)-r$ ，其中 $\min(l)$

表示后缀 $\min$。那么我可以在右边跑一个双指针维护一下，开一些键值为 $\max(r)-r$ 桶计算。

考虑将每个区间表示为二维平面上的点 $(x,y)$
首先先初始化 $(x,y)$ 的权值为 $y-x$ ：对 $x=1…n$ 进行矩形减，对$y=1…n$ 进行矩形加，即 $[x,1..n]-=x，[1..n,y]+=y$
然后对序列进行最值分治，每次分治相当于一个矩形内的点对应的序列上的区间的 $\max$ 都是分治中心，进行矩形加，$\min$ 同理进行矩形减

问：

给你一个排列，求极差为 $x$ 的区间的最值。

给你一个排列，求极差为 $x$ 的区间的最长长度。

给你一个排列，求极差为 $x$ 的区间的个数。

CF464E The Classic Problem

这里难点就是会爆 long long。所以我们用线段树代替堆。

当然要可持久化一下，不然会爆空间。

加法就是区间赋 $0$，比大小主席树上 hash + 二分即可。

CF603E Pastoral Oddities

首先有一个结论：满足存在子图使得每个点度数都为奇数的充要条件是这个连通块点数为偶数。

证明：最开始所有点的值都为偶数（$0$），而每加一条边进入使用集合，都不会改变每个点的度数之和的奇偶性，也就是说，每个点的度数之和总是膜 $2$ 余 $0$，因此结论成立。

然后两个联通块合并也不会使奇数块增加。

想到 kruskal。考虑一条边被用的时间，肯定要么不被用上，要么是从加入开始的一段连续的区间。

考虑逆向思维，现分治右儿子，维护出最大边权 $w$。

然后把其他大于 $w$ 的边加入。（因为他们是被 $w$ 踢掉的）

说详细些，就是先把线段树上一个点存的所有边都加进来，然后先递归右儿子，再遍历左儿子，遍历完两个子树后就把属于这个节点的被标记的边删掉。当我遍历到叶子节点时，我就一个一个从小到大加边，（用堆维护总复杂度 $O(n\log ^2n)$）只要联通了就不加，那么答案就是我最后加入的那一条边的权值。对于其他点，相当于在这个时候我就已经不用了，打上标记上传。

CF1446D Frequency Problem

首先有一个结论：两个众数之一一定是全局众数 $Z$。

因为假设我们找到一个区间，它包含的两个众数都不是全局众数，那我肯定可以向两边拓展。

然后对于 $D1$，我们已经可以做了。对于每个值都扫描一遍序列，可以把 $Z$ 设为 $+1$，当前枚举的值设为 $-1$，那么就只要找极长的和为 $0$ 的段。这个我们可以开一些桶直接扫一遍。

对于 $D2$ ，考虑根号分治。

对于出现次数 $\ge \sqrt n$ 的数，总共不超过 $\sqrt n$ 个，直接采用 $D1$ 的做法。复杂度 $O(n\sqrt n)$。

剩下的就都是出现次数 $\le \sqrt n$ 的数了。那么答案包含的区间内众数最多也只出现了 $\sqrt n$ 次。那么考虑枚举众数出现的次数 $c$ ，然后使用双指针，满足区间内没有数出现次数 $>c$，扫一遍就好了。
