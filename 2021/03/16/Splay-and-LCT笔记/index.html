Splay-and-LCT笔记

2021-03-16
#数据结构

Splay 其实几年前就学过了，但是一直觉得太长了很少写就忘了。。。

LCT 也因此咕了。

Splay 复习

一些定义：

int root, tot; //根，总结点个数
struct Splay
{
    int fa, ch[2]; //父亲，左右儿子
    int val, cnt, siz; //这个点的权值，重叠了多少个点，子树大小 
} tree[N];

三种基础操作：

void Maintain(int x) { tree[x].siz = tree[tree[x].ch[0]].siz + tree[tree[x].ch[1]].siz + 1; } //维护 siz

bool ID(int x) { return x == tree[tree[x].fa].ch[1]; } //返回是左儿子还是右儿子

void Clear(int x) { tree[x] = { 0, { 0, 0 }, 0, 0, 0 }; } //销毁节点

然后是旋转：

void Rotate(int x)
{
    int y = tree[x].fa, z = tree[y].fa, id = ID(x);
    
    tree[y].ch[id] = tree[x].ch[id ^ 1];
    if(tree[x].ch[id ^ 1]) tree[tree[x].ch[id ^ 1]].fa = y; //把 x 的儿子和父亲对接
    
    tree[x].ch[id ^ 1] = y;
    tree[y].fa = x; //把 x 设为它父亲的父亲
    
    tree[x].fa = z;
    if(z) tree[z].ch[y == tree[z].ch[1]] = x; //如果还有爷爷，设定父子关系。
    //注意这里不要用 ID(y)，因为 y 的父亲已经设成 x 了
    Maintain(x), Maintain(y);
    return;
}

这里也不用多讲。

Splay

把一个点转到根。

记得写双旋，不要写 $Spaly!!!$

之前以为双旋是转两下就转到根，觉得好神奇。。。

其实是均摊 $O(\log n)$ 的。

void Splay(int x)
{
    for(int f = tree[x].fa; f; Rotate(x), f = tree[x].fa) //每次都转
    	if(f != rt) Rotate(ID(x) == ID(f) ? f : x); //没有爷爷了只单旋，否则分类讨论先旋 x 还是它父亲
    root = x; //记得改 root
}

那么主要部分就没有了，下面就是配套实现。

插入

设插入元素权值为 $v$。

  1. 根节点为空，直接插入。

  2. 找到一个点 $x$，$tree[x].val=v$，那么更新 $cnt$，并 $Splay$。

  3. 否则二叉搜索树递归，找到空节点插入并将其旋转至根。

void Insert(int v)
{
    if(!root)
    {
        tree[++tot] = (Splay) { 0, { 0, 0 }, v, 1, 1 };
        return;
    }
    int x = root, f = 0;
    while(1)
    {
        if(tree[x].val == v)
        {
            ++tree[x].cnt;
            Maintain(x), Maintain(f);
           	Splay(x);
            return;
        }
        f = x, x = tree[x].ch[tree[x].val < v];
        if(!x)
        {
            tree[++tot] = (Splay) { f, { 0, 0 }, v, 1, 1 };
            tree[f].ch[tree[f].val < v] = tot;
            Maintain(tot), Maintain(f);
            Splay(tot);
            return;
        }
    }
    return;
}

合并

把一棵树的最大值转到根，再把它右儿子设为另外一棵树的根。

LCT

解决动态树问题。

辅助树

LCT 是实链剖分的，一个非叶点有且仅有一个实儿子，由实边构成的极长路径称为一条实链，每条实链用一棵 Splay 维护，其中序遍历就是实链上所有点深度从低到高的顺序。

注意每棵 Splay 的根指向这条实链的父节点，也就是说父亲指针不一定与原树相同。

access

access(x) 把点 $x$ 到根的路径放在一棵 Splay 里。

思路是从底向上更新，每次把当前节点的父亲转到 Splay 的根，然后删除与右儿子的边，把右儿子变成当前节点，并更新信息。

int access(int x)
{
	int p = 0;
	for(; x; p = x, x = f[x])
	{
		Splay(x);
        ch[x][1] = p;
        pushup(x);
	}
    return p;
}

makeroot

换根。

就是 access(x) 然后把这棵 Splay 取反。

Link

先 makeroot(x)，然后把 f[x] 设为 y。

Cut

保证合法，就 makeroot(x)，再。。。

Find

找树根。

先 access 一下，再 Splay 一下，一边 pushdown 一边走左儿子就好了。

可以维护连通性。
