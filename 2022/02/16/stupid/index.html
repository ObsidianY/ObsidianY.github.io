养傻记

2022-02-16

手持两把锟斤拷， 口中疾呼烫烫烫， 脚踏千朵屯屯屯， 笑看万物锘锘锘

O(n)-O(1) gcd!!!

$O(n)$ 预处理，$O(1)$ 查询值域在 $[1,n]$ 之间的每个整数的 $\gcd$。

直接把每个数拆成三个数的乘积，那么这三个数可以满足：

  • 要么都 $\le \sqrt n$

  • 要么有一个是 $>\sqrt n$ 的质数，其他两个 $\le \sqrt n$

怎么把一个数拆成这样三个数的乘积呢？我们可以线性筛出所有数 $x$ 的最小质因子，设为 $p_x$，对于 $\frac{x}{p_x}$ 的分解设为 $(a,b,c)$。

那么我们只要找到 $\min\{a,b,c\}$，把 $p_x$ 丢进去就可以了。

证明就是大力分讨，如果不成立就会爆上界。

然后预处理 $1\sim\sqrt n$ 这些数两两的 $\gcd$，这个直接递推即可。

对于查询 $\gcd(x,y)$，把 $x$ 的分解拿出来，用预处理出的 $\sqrt n$ 以内的 $\gcd$ 直接算出相同因数，而对于分解出的大质数直接特判即可。

CF1628D Game on Sum

看起来很高妙的博弈？其实是真傻了！

盯着题面发呆良久后发现，woc！Easy Version 可以直接 DP！

直接写出：$dp[i][j]$：操作 $i$ 次，至少加 $j$ 次的答案。

则 $dp[i][0]=0,dp[i][i]=ki$。

而对于剩下的情况，设选的是 $t$，则 $dp[i][j]=\min(dp[i-1][j]-t+dp[i-1][j-1]+t)$。显然最优解为 $\frac{dp[i-1][j]+dp[i-1][j-1]}2$。

这个 $k$ 只是个比例，可以直接去掉，这样就可以过 Easy Version 了。

再来看看这个递推的形式。不妨把它化一下，设 $dp[i][j]$ 表示 $i$ 次加，$j$ 次减的答案。边界条件是 $dp[i][0]=i,dp[0][i]=0$。

那个转移式即为 $dp[i][j]=\frac{dp[i-1][j]+dp[i][j-1]}2$。
嘿！这是一个非常好看的二维递推形式。那么答案就是 $x$ 轴上的一些点，向上和向右，走到 $(m,n-m)$ 的方案数之和。当然，我们要把 $(i,0)$ 的答案乘上 $2^{i-n}i$。

形式化地，答案为 $k\sum\limits_{i=0}^m 2^{i-n}i\binom{n-i-1}{m-i}$ 。这里是 $n-i-1$ 的原因是一个点开局只能向上走。

麻了，Hard Version 没看到 $sum\le 1e6$……

以为 DP 没用，然而思考暴力方法往往对正解的思考大有裨益。

CF1559E Mocha and Stars

这都不会，可以cs了！

忽略 $\gcd$ 条件，哇！我会 DP！设 $dp[i][j]$ 为前 $i$ 个数，和为 $j$，则有 $dp[i][j]=\sum\limits_{k=l_i}^{\min(r_i,j)}dp[i-1][j-k]$。 前缀和优化即可做到 $O(nm)$！

然后我们发现这个 $\gcd$ 看着真的太难受了，于是设 $f(m,a_1,a_2,\ldots,a_n)$ 为一种拆分方法是否满足除 $\gcd$ 以外的其他条件。最终答案可以写成 $\sum\limits_{a_1=l_1}^{r_1}\sum\limits_{a_2=l_2}^{r_2}\cdots\sum\limits_{a_n=l_n}^{r_n}[\gcd(a_1,a_2,\ldots,a_n)=1]f(m,a_1,a_2,\ldots,a_n)$。

哦，这不就是我们亲爱的 Möbius 反演的形式吗？

后面那个东西，就是可以 $O(\frac{nm}d)$ 直接算的啦！总复杂度 $O(nm\log m)$。

难处理的部分拆出去，用常见套路处理掉。

CF1542D Priority Queue

直接做完全不会，考虑算贡献。

这个贡献是可以 DP 的。考虑当前我们要算位置 $p$ 留到最后的方案数，设 $dp[i][j]$ 表示经过前 $i$ 个操作之后，还剩下了 $j$ 个比 $a_p$ 小的数的方案数。这里对于与 $a_p$ 相同的数的处理方式是：钦定优先队列中值相同的数，先进先出。即把 $a_x=a_p$ 而 $x<p$ 的数视为 $<a_p$。

转移方程显然。注意在 - 且 $i<p$ 的位置，$dp[i][0]=2\times dp[i-1][0]+dp[i-1][1]$。

拆贡献！！！

$O(n^2)$ DP：

一类相邻型DP的套路

例题：[CEOI2016]kangaroo

题意：给你 $n,s,t$，求 $\sum\limits_{P}[p_1=s][p_n=t]\prod\limits_{i=2}^{n-1}[[p_i<p_{i-1}]=[p_i<p_{i+1}]]$。

其中 $P$ 为长度为 $n$ 的排列。

考虑维护连续段。设 $dp[i][j]$ 表示 $1\sim i$ 中所有数排成一些连续段，考虑段间顺序，其中每个连续段中的数 $a_i$（两端除外）满足它要么同时小于左右两个数，要么大于左右两个数。

插入有两种情况:

  1. 合并相邻的两段。由于从小到大插入，这样一定是合法的。

  2. 加入一个新段。

其他细节就随便啦。

dp[1][1] = 1;
For(i, 2, n) For(j, 1, i)
{
	if(i != s && i != t)
    dp[i][j] = Add(Mul(dp[i - 1][j - 1], (j - (i > s) - (i > t))), Mul(dp[i - 1][j + 1], j));
	else dp[i][j] = Add(dp[i - 1][j - 1], dp[i - 1][j]);
}

回到本题。设 $dp[i][j]$ 表示长度为开了 $i$ 台，连出了 $j$ 个连续段的方案数。当然，这些连续段之间空的距离至少为 $2$。

然后也是大力分类就是 $O(n^2)$ 的了。

dp[0][0] = 1;
For(i, 1, n) For(j, 0, i)
{
	if(j) dp[i][j] = Add(dp[i][j], Mul(j, dp[i - 1][j - 1]));
	dp[i][j] = Add(dp[i][j], Mul(dp[i - 1][j], 2 * j));
	if(i > 1)
	{
		dp[i][j] = Add(dp[i][j], Mul(dp[i - 2][j], 2 * j));
		dp[i][j] = Add(dp[i][j], Mul(2 * j, dp[i - 2][j + 1]));
	}
	if(i > 2) dp[i][j] = Add(dp[i][j], Mul(j, dp[i - 3][j + 1]));
}

其实 $m$ 才是序列长度，所以我们可以考虑把一个连续段压成一个长度为 $1$ 的段。也就是说，只在颜色的交界处进行转移。原因是我总不可能在一种颜色条的内部切一段吧。。。

CF241B Friends

大家好呀，我做过异或粽子，是 $O((n+k)\log (nV))$ 的

但是本题 $k$ 有点大。。。有个 $O(n(\log^2V+\log n))$ 的与 $k$ 无关的做法！

（其实去年做异或粽子就看到这题了，然后就不会了）

首先第 $k$ 相关的第一思路：binary search!

直接对于每个数在 trie 上算就可以了。$O(n\log^2 V)$。

现在找到了第 $k$ 大的值。查询时，提前把 $a$ 排序，然后 trie 上的一个子树就是原序列的一个区间。子树求异或和就只要拆位前缀预处理即可。

求第 $k$ 大我为什么会先想到堆呢。。。

考虑顺着扫，扫到一个点就把它对应的值打个访问标记。

那么由于是等差数列，状态合法当且仅对于当这个位置对应的值 $a_i$，存在 $k$ 使得 $a_i-k$ 的状态与 $a_i+k$ 的不同。即不存在合法解当且仅当以 $i$ 为中心是回文的。权值线段树维护即可。

CF860E Arkady and a Nobody-men

$O(n\log^2 n)$ 做法：

考虑维护答案的增量。

$g(u)=g(fa_u)+\sum\limits_{dep(u)=dep(v)}{dep(\mathrm{lca}(u,v))}-1$。

分别处理深度相同的点，祖先加祖先求和即可。

$O(n\log n)$ 做法：

把点按照 $dfn$ 排列后，发现一个点对于它同层的前面的点，$lca$ 的深度是单调不减的。

维护一个栈表示当前点与前面的 $lca$ 的序列，插入一个点 $u$ 时，如果最后一个和倒数第二个点与 $u$ 的 $lca$ 相同，则合并它们。显然它们之后与其他点的 $lca$ 也会是一样的了。然后正着反着做一遍即可。

好像有 $O(n)$ 长剖做法，以后补补。

CF1044F DFS

手玩发现，加入一条边，会使得在它的两个端点内侧的所有点变得不合法。用另一种方式描述是，我们删掉这两个端点 $u,v$ ，形成的若干个联通块中，原来与 $u,v$ 都相邻的联通块是不合法的。

为什么呢？考虑一张图，它的 $dfs$ 生成树一定是只有树边和返祖边的。所以对于一条边 $(u,v)$ ，以它对应的合法起始点为根，$u,v$ 之间一定是祖先后代关系。

那么直接 $dfs$ 一遍，不合法的区间只有 $O(1)$ 个，用线段树维护区间 $min$ 以及 $min$ 的个数即可。
